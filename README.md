# goes-security-skill-vue

Claude skill for generating **professional-grade security tests** for Vue 3 + Vitest + Playwright projects, with a Custom HTML + Excel Reporter (no Java, no Allure).

Covers the **GOES Cybersecurity Checklist** (frontend items), **OWASP Top 10** and Vue-specific controls (VF1-VF12). Goes beyond test code: applies a series of **hardening practices to the project itself** (no-emojis policy, login card design, ThemeToggle variant, paginated catalogs, brand centralization, ESLint ignores for vendored reporter, Husky guardrails, etc.) — every rule was learned auditing a real MINEDUCYT SPA.

---

## What you get when you activate the skill

When Claude reads this skill in a Vue project, it:

1. **Analyzes the actual code** — stores, composables, router, interceptors, components.
2. **Installs the dev deps** that are missing (`vitest`, `@vue/test-utils`, `jsdom`, `xlsx`, optionally `@playwright/test`).
3. **Configures Vitest + Playwright** with the bundled reporter (no file duplication — references it directly from `.claude/`).
4. **Generates 16 patterns of security specs** (13 unit + 3 E2E) covering GOES R3-R55 + VF1-VF12 + OWASP A01-A07-A09.
5. **Documents out-of-scope controls** in `not-applicable.security-html.spec.ts` (password hashing, rate limiting, file uploads when none exist, SQLi, CORS, DB encryption, security headers, webhooks, PCI).
6. **Produces a professional README** for the project (RBAC matrix, env vars, scripts, seeded credentials, suite-of-security table).
7. **Runs the suite** and produces:
   - `reports/security/security-report.html` — interactive report with sidebar Epic > Feature > Story, modal with **file path** of each spec, charts SVG, dark theme, buckets (Pasados / Desactivados / Migrados / No aplicables)
   - `reports/security/security-report.xlsx` — workbook with one sheet per bucket, ready as audit annex

Each test ships:

- **Input evidence** — attacker payload / scenario as JSON
- **Output evidence** — defense result / store state after the action
- **Step narrative** — `Prepare / Execute / Verify` so the auditor can follow the test
- **OWASP doc link** — `<a href="https://owasp.org/Top10/A0X_2021-..." target="_blank">` in `descriptionHtml`

This is what a single test looks like when generated by the skill (excerpt):

```typescript
it('[R9] clearSession wipes accessToken, user, roles and timestamps', async () => {
  const t = report()
  t.epic('Session Management')
  t.feature('Logout hygiene')
  t.story('clearSession wipes ALL auth state')
  t.severity('critical')
  t.tag('Pentest', 'OWASP-A07', 'GOES-R9')
  t.descriptionHtml(
    '<p>A logout must leave the store identical to its initial state...</p>' +
    '<p><strong>Reference:</strong> ' +
    '<a href="https://owasp.org/Top10/A07_2021-..." target="_blank" rel="noopener">OWASP A07</a>.</p>',
  )

  t.step('Prepare: populate the store with a fully authenticated session')
  const auth = useAuthStore()
  const authenticated = session()
  auth.setSession(authenticated)
  t.evidence('Input - session before logout', {
    accessToken: authenticated.accessToken,
    user: authenticated.user,
    isAuthenticated: auth.isAuthenticated,
  })

  t.step('Execute: call auth.clearSession()')
  auth.clearSession()

  t.step('Verify: every sensitive slot is back to its initial value')
  expect(auth.isAuthenticated).toBe(false)
  expect(auth.accessToken).toBeNull()
  // ... 4 more assertions

  t.evidence('Output - store snapshot after clearSession', {
    accessToken: auth.accessToken,
    user: auth.user,
    roles: auth.roles,
    isAuthenticated: auth.isAuthenticated,
  })

  await t.flush()
})
```

```bash

cp -r goes-security-skill-vue/.claude/ /path/to/your-vue-project/.claude/
```

## What the skill applies beyond tests

Auditing a real SPA taught us that good security testing alone is not enough — the surrounding project conventions matter. The skill enforces these **hardening rules** when generating / reviewing code (full detail in SKILL.md PASO 8):

| Rule | Why |
|------|-----|
| **No emoji icons — use Lucide** | Emojis render inconsistently across OS; Lucide inherits the theme color and scales as vectors. |
| **Login: card design + show-password + dark mode tokens** | The default Vite scaffold is too plain; auditors expect a polished entry point. |
| **`ThemeToggle` with `variant` prop** (`topbar` \| `default`) | One palette doesn't fit both navy topbars and surface headers — the toggle must be visible everywhere. |
| **Paginated catalogs** (search + page-size, client-side for small ones) | < few hundred rows: client-side. Larger: server-side via `?page=&limit=&search=`. |
| **`/auth/logout` is public on the BE** | A user with an expired access token must still be able to log out cleanly — guard would brick them. |
| **`@ts-expect-error` always with description ≥3 chars** | Otherwise lint-staged aborts the commit (`ban-ts-comment` rule). |
| **`tests/security/reporter/**` excluded from ESLint** | Vendored 3rd party — do not lint vendored code. |
| **`/reports` in `.gitignore`** | Generated artefact, regenerated each run. |
| **Brand centralization** | Constant in `@/app/config/branding.ts` so rebrands are one-line. |
| **Exhaustive N/A spec** | Auditor copies/pastes each row from the Excel directly into the regulatory annex — silence is not an option. |
| **`descriptionHtml` always with OWASP link** | The HTML report links directly to canonical control documentation. |

## View the report

```bash
npm run test:security
# HTML:  reports/security/security-report.html  (opens directly in any browser)
# Excel: reports/security/security-report.xlsx  (one sheet per bucket)
```

E2E (requires backend running on `/api`):

```bash
npx playwright install   # one-time, ~150 MB
npm run test:e2e
```

Both files are self-contained and regenerated on every run. The HTML is ideal for local exploration; the Excel is ideal as an annex for security / compliance reviews.

The HTML modal now shows the **spec file path** under each test title (📄 `tests/security/<file>.spec.ts`) so an auditor can jump directly to the source of any assertion.

## Requirements

- **Vue 3** project with Vitest configured (or Claude installs it)
- **Node.js 18+**
- **No Java required** — pure Node.js HTML + Excel reporter
- **`xlsx` (SheetJS)** as devDependency (the skill installs it automatically)
- **Playwright** optional — only required for patterns 10-12 (E2E). Run `npx playwright install` once after `npm install`.
